视联浏览器双机热备的工作原理

主机上视联浏览器服务端中间件（后续均简称为中间件）正常运行，被机运行热备程序，当主备之间心跳超时，热备程序通过脚本服务自动拉起备机的中间件程序

注：必须找到一个主机和备机均能入视联网的mac（类似互联网的ip，主要作用是入视联网并且进行相关视联网业务）

热备程序的具体实现

热备程序通过调用 v2v_drv.h （视联浏览器走视联网的核心服务均通过调用 v2v_drv 可执行文件完成）提供的 v2v_echo 接口 来进行判断主机的中间件是否正常运行，若正常，则什么都不做，若不正常，则连续计时8次仍不正常则认为主机故障，通过脚本启动备机的中间件

v2v_echo 接口说明

int v2v_echo(const char *dst_number, char *voice, int len/*max is V2V_MAX_DATA_LEN-28*/, int timeout_ms, int *delay_us);
第一个参数 dst_number 目标的20位号码
第二个参数 voice 回声的数据, 随便来个buff
第三个参数 len V2V_MAX_DATA_LEN-28
第四个参数 timeout_ms 最大超时时间 基本单位 毫秒
第五个参数 delay_us 延时时长 基本单位 微妙
