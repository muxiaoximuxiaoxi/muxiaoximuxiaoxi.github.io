Windows 客户端的自动化编译

配置节点

Windows 需要通过 Java Web 启动 Jenkins节点的代理
下载安装 Java 工具 并配置环境变量

jdk17官方下载

下载之后, 一路默认安装或者自定义安装, 但需要记下安装路径

根据节点提示信息, 在目标Windows用cmd命令行执行命令
curl -sO http://10.2.14.48:8080/jnlpJars/agent.jar

java -jar agent.jar -jnlpUrl http://10.2.14.48:8080/manage/computer/dzp%2Dwindows/jenkins-agent.jnlp -secret acde60bc0235357b78b0a5fa47111e6a73e8ae8a5d5bf46c1f37c25bec31d59d -workDir "D:\jenkins"

Windows 系统配置节点开机自启动

配置 .vbs文件
rem wscript是一个脚本宿主文件, 提供了解析以上语句的功能, 也叫脚本解析引擎
rem 这句话的意思就是说, 创建一个脚本命令窗口
set ws=WScript.CreateObject("WScript.Shell") 
ws.Run "D:\svn_code_shh\run.bat /start",0
配置 .bat脚本，内容如下
rem 由于是使用 bat 脚本打开cmd，执行多条语句时，需要使用 "" 及 &&
rem cmd /k 表示用cmd运行命令后不关闭。
rem 切换到指定目录 且 执行 相应的java语句 (Jenkins 相应节点配置提示信息)

cmd /k "cd /d D:\svn_code_shh&&java -jar agent.jar -jnlpUrl http://10.2.14.48:8080/computer/shh%%2Dwindows/jenkins-agent.jnlp -secret @secret-file -workDir "D:\svn_code_shh""
将.vbs文件放到C:\ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp目录下，开机会自动运行.vbs文件

将.bat文件放到配置的路径下，重启测试。

Windows SSH 服务端及客户端安装

Windows设置—–> 应用 —–> 可选功能—–> 添加功能 —–>SSH—–>安装OpenSSH服务端 —–>安装OpenSSH客户端

注：此cmd窗口需要以管理员身份运行

[配置开机自启sshd服务]
sc config sshd start= auto

[启动OpenSSH服务]
net start sshd

[SSH连接]
ssh 用户@ip地址
输入用户密码

通过预置密钥对，开启免密登录

ssh出于安全考虑，每次连接都会提示输入密码，为了避免麻烦，就要配置密钥对。

生成密钥对

ssh-keygen -t rsa

将公钥(id_rsa.pub)传到远程主机

使用Xshell的Xftp进行远程传输

将公钥(id_rsa.pub)复制到远程主机的~/.ssh目录下，并重命名文件名为authorized_keys

修改远程主机sshd_config配置

vim /etc/ssh/sshd_config
PubkeyAuthentication yes
StrictModes no

重启sshd服务

systemctl restart sshd

cmd连接远程主机

ssh 用户@ip地址

构建触发器 —–> 轮询 SCM-日程表

轮询规则表
字段 * * * * *
含义 分钟 小时 日期 月份 星期
取值范围 0~59 0~23 1~31 1~12 0~7
示例
每隔15分钟执行一次 H/15 * * * *
每隔2个小时执行一次 H H/2 * * *
每隔3天执行一次 H H H/3 * *
每隔3天执行一次（每月的1~15号） H H 1-15/3 * *
每周1,3,5执行一次 H H * * 1,3,5

命令及解释
