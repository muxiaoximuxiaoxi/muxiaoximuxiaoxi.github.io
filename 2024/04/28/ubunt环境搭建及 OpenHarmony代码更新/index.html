二、Ubunt设置samba服务

1、开启samba服务器
sudo apt-get install samba samba-common

2、下载samba并配置：

#下载samba
sudo apt-get install samba
sudo apt-get install smbclient
#修改配置文件（缺少vim可用vi，也可以apt-get install vim）
sudo vim /etc/samba/smb.conf

执行之后如需遇到：vim: command not found
则说明还未安装vim，直接输入sudo apt install vim
即可进行安装

安装vim之后，重新执行：sudo vim /etc/samba/smb.conf
在配置文件smb.conf的最后添加下面的内容：

[root]
comment = All users
force user = root
path = /root
browseable = yes
available = yes
public = yes
writeable = yes

⑥重启samba服务器。
sudo service smbd restart

2、开启samba客户端
Win10打开控制面板–程序–windows功能启动或关闭–打开smb客户端

开启samba客户端
Win10打开控制面板–程序–windows功能启动或关闭–打开smb客户端
关闭虚拟机，重启windows

3、访问共享目录
①获取到linux系统的IP，在linux下输入
Ifconfig
如出现：Command ‘ifconfig’ not found, but can be installed with:
直接执行：sudo apt install net-tools
然后在执行：ifconfig

②Windows下添加linux的IP地址
Windows徽标+R 在弹出的运行窗口中输入 \ip 即可访问。如\192.168.182.188,输入samba用户名及密码访问即可看到共享，然后就可以在Linux系统与Windows系统直接进行文件共享了

三、配置root用户

1、直接执行：sudo passwd root
2、设置root用户密码即可
3、执行：su
切换root用户

四、安装git及配置git用户信息

1、执行：sudo apt install git-all

2、执行：curl -s https://packagecloud.io/install/repositories/github/git-lfs/script.deb.sh | sudo bash

3、执行：sudo apt-get install git-lfs

4、执行：git lfs install

5、配置用户信息

git config –global user.name “yourname”
git config –global user.email “your-email-address”
git config –global credential.helper store

6、配置gitee SSH公钥
①执行：ssh-keygen -t ed25519 -C “xxxxx@xxxxx.com“
直接一路回车下去

②执行vim ~/.ssh/id_ed25519.pub
复制内容即可

③粘贴至gitee相关位置

五、拉代码

1、切换至linux和windows的共享文件夹

2、安装repo工具（如已安装，请略过此节）
①执行： curl https://gitee.com/oschina/repo/raw/fork_flow/repo-py3 > /usr/local/bin/repo
如出现：Command ‘curl’ not found, but can be installed with
直接执行：apt install curl
继续执行： curl https://gitee.com/oschina/repo/raw/fork_flow/repo-py3 > /usr/local/bin/repo

②执行：chmod a+x /usr/local/bin/repo
③执行： pip3 install -i https://pypi.tuna.tsinghua.edu.cn/simple requests
如遇到：Command ‘pip3’ not found, but can be installed with
直接执行：apt install python3-pip

3、执行：repo init -u git@gitee.com:openharmony/manifest.git -b master –no-repo-verify
如果出现：/usr/bin/env: ‘python’: No such file or directory
直接执行：sudo apt-get install python3.4

继续执行：sudo ln -s /usr/bin/python3 /usr/bin/python
继续执行：repo init -u git@gitee.com:openharmony/manifest.git -b master –no-repo-verify

4、执行：repo sync -c

5、执行：repo forall -c ‘git lfs pull’

六、编译

1、安装编译环境
①执行：sudo apt-get update && sudo apt-get install binutils git git-lfs gnupg flex bison gperf build-essential zip curl zlib1g-dev gcc-multilib g++-multilib libc6-dev-i386 lib32ncurses5-dev x11proto-core-dev libx11-dev genext2fs liblz4-tool libssl-dev libtinfo5 lib32z1-dev ccache libgl1-mesa-dev libxml2-utils xsltproc unzip m4 bc gnutls-bin python3.8 python3-pip ruby

②执行：bash build/prebuilts_download.sh

③执行：./build.sh –product-name rk3568 –ccache

如出现：FileNotFoundError: [Errno 2] No such file or directory: ‘java’

直接执行：sudo apt install openjdk-17-jre-headless

继续执行：./build.sh –product-name rk3568 –ccache

看到：buidl successful 说明编译通过
