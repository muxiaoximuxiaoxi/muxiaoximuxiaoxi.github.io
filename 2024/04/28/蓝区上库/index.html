主干上库

新建issue (issue建在公共仓)

将需要上库的仓库fork到自己的私仓或者将自己私仓更新

git clone + ssh

正常修改代码

git add .

git commit -sm + “issue名”

git push

新建pr (pr建在私仓)

当发现有多次提交多次修改的时候

但该库管理员要求只能有一条提交时，或者给了好看最好只有一条提交

git reset HEAD~n 回到上…上（n个上）版本

git push -f origin master （-f 强推）


使用一次
git reset HEAD3 回到上上上（3个上）版本
git push -f origin master （-f 强推）
使用一次
git reset HEAD1 回到上（1个上）版本
git push -f origin master （-f 强推）

此时库上已没有提交记录


正常上库流程
git add .
git commit -sm + “issue名”
git push


此时也只有一次提交记录

跑门禁（每个仓的跑门禁指令各不相同，具体根据仓而定）

sig start build

分支上库

新建issue (issue建在公共仓)

将需要上库的仓库fork到自己的私仓或者将自己私仓更新

git clone + ssh

将私仓代码拉至虚拟机

git branch -a

查看当前所有分支信息

git checkout +分支名

切换所在分支

git branch -a

查看是否已切换分支

正常修改代码

git add .

git commit -sm + “issue名”

git push

新建pr (pr建在私仓)

pr 开始跑门禁

start build
